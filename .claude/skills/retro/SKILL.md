# retro

**회고 고도**의 스킬이다. 하나의 비자명한 작업이 *끝난 뒤*, 그 작업을 정직하게 평가하고 **재사용 가능한 학습을 durable 표면에 남긴다.**

산출물은 성찰 산문이 아니라 **레포/메모리에 실제로 남는 변경**이다 — CLAUDE.md 규칙 한 줄, ADR, memory 파일. 증발하는 회고는 토큰 낭비다. 학습이 없으면 "없음"을 한 줄로 남기고 끝낸다(억지로 짜내지 않는다).

> 이 스킬의 순이익은 **여러 작업에 걸친 반복 패턴**과 **프로세스 결함**을 잡는 데 있다. 단건 결정의 근거(ADR)·단건 사실(memory)은 이미 다른 메커니즘이 잡는다. 회고는 그것들이 못 보는 "이번에도 또 그랬다"를 잡아 승격한다.

인자(선택): 회고 대상 작업의 한 줄 요지 (예: `/retro ADR-0006 감사 sink`). 없으면 직전 완료 작업을 대상으로 한다.

## 0. 돌릴까 말까 (반의례 게이트)

회고가 매번 도는 순간 의례가 되어 진짜 학습을 noise로 묻는다. 먼저 판정한다:

- **돌린다**: 기능·버그·리팩터·설계 등 비자명·다단계 작업. 리뷰/테스트/현실이 뭔가 잡았거나, 사용자가 정정했거나, 예상과 어긋난 지점이 있었던 작업.
- **건너뛴다(그냥 종료)**: typo·설정 변경·한 줄 수정처럼 배울 게 구조적으로 없는 작업. `회고 skip(trivial)` 한 줄만 남기고 끝.

애매하면 §1을 빠르게 훑어 학습 씨앗이 있는지 보고, 없으면 즉시 skip.

## 1. 증거 수집 (무슨 일이 실제로 있었나)

정직하게 되짚는다 — 잘된 얘기 말고 **마찰**을 찾는다:

- **결과물이 처음에 틀렸던 지점**: 리뷰(codex/advisor)·테스트·현실이 잡았는데 self-review는 놓친 것. 몇 라운드 걸렸나.
- **사용자 정정**: 사용자가 방향을 바꾸거나 놓친 걸 지적한 지점 — 그게 가장 강한 재사용 신호다.
- **헛돈 지점**: 막다른 길, 반복한 수정, 잘못한 가정, 놓친 도구/채널/명령.
- **놀란 점**: API·도구·환경이 예상과 다르게 군 곳.

> **서브에이전트 위임 작업 주의(구조적 맹점)**: dispatch-issue처럼 구현을 서브에이전트에 위임한 경우, 너는 **서브에이전트의 최종 반환만** 보고 그 transcript(thrash·red-green churn)는 못 본다. 그 반환의 "마찰·판단·codex 대응" 항목을 증거로 쓰되, **보이지 않는 것을 추측하지 말고** 회고를 오케스트레이션 수준 + 반환된 마찰로 명시적으로 한정한다. (그래서 `go-tdd-implementer`는 최종 반환에 마찰 한 줄을 남긴다 — 그걸 소비한다.)

## 2. 2축 평가

① **결과물 품질** — 견고했나? 리뷰/테스트가 잡은 게 self-review로 잡혔어야 했나? (예: 어떤 ADR이 codex 재검증 3라운드가 필요했다면, self-review 캘리브레이션이 낮았다는 신호다.)
② **프로세스** — 어디서 헛돌았나? 놓친 채널·잘못한 가정·반복 정정.

**self-grade는 후하다.** 외부 평가자(advisor/codex)를 부를 땐 **무엇을 평가시키는지 구분한다** — 이 둘은 다르다:

- **결과물 자체 평가**: 그 작업이 *이미* 외부 리뷰(codex 3라운드 등)를 거쳤다면 **다시 돌리지 않는다** — 그건 의례다. 리뷰 기록을 §1 증거로 읽으면 된다.
- **회고 결론 평가**: 학습의 프레이밍·라우팅이 과/소하지 않은지. 고-stakes이거나 결과물이 크게 흔들렸으면 **이쪽을** 외부에 검증받는다(첫 스킬은 성공처럼 보이려 패턴을 실제보다 조이는 confirmation-bias가 있다 — 그걸 외부가 깬다).

저-stakes면 self로 충분하다(외부 호출은 비용·의례이므로 남발하지 않는다).

## 3. 재발 점검

이 학습이 **처음인가, 또인가.** 기존 memory(특히 `type: feedback`)를 훑어 같은 교훈이 이미 있는지 본다. memory 디렉터리는 세션마다 `<system-reminder>`의 MEMORY.md 경로로 주어진다:

```bash
grep -ril "<키워드>" "<memory-dir>"     # 같은 교훈이 이미 파일로 있나
```

- **처음** → §4로 라우팅.
- **또(재발)** → 승격 신호. 같은 교훈이 실제로 반복됐으면 memory 노트를 **규칙(CLAUDE.md)**으로 올리거나 훅/스킬 변경을 검토한다(승격 바는 §4).

> 이건 `/dream`(memory-dream)의 통합 작업의 **얇은 사촌**이다 — 메모리 전반 재구성·중복 제거는 `/dream`에 맡기고, 여기선 "이 작업의 학습이 기존과 겹치나"만 가볍게 본다. 재발이 쌓였다 싶으면 `/dream`을 권한다.

## 4. 표면 라우팅 (핵심) — 학습은 *실행 가능*해야 한다

라우팅만으로 부족하다. **캡처는 실행 가능해야 한다 — 명령·규칙·체크지, 훈계가 아니다.** "codex 코멘트 잘 보기"는 증발한 산문과 다를 바 없다. 다음에 그대로 실행/적용할 수 있어야 살아남는다:

**Bad(훈계, 죽음)**:
> codex 리뷰를 꼼꼼히 확인하기

**Good(실행 가능, 생존)**:
> PR 후 CLI 출력뿐 아니라 GitHub 봇 코멘트도 확인:
> `gh api repos/<owner>/<repo>/pulls/<N>/comments --jq '.[] | {path,line,body}'`

§4의 품질 게이트: 캡처가 **명령/규칙/체크**가 아니면 다시 쓴다.

라우팅 표(학습 1개당 표면 1개):

| 학습 종류 | 표면 | 예 |
|---|---|---|
| 항상 켜둘 재사용 규칙 | **CLAUDE.md** (아래 승격 바 통과 시만) | "PR 후 GitHub 봇 코멘트 확인" |
| 아키텍처 결정의 why·버린 대안 | **ADR**(`docs/adr/`) 또는 `/architect` 안내 | "내구성 계약은 Accepted 전 adversarial 패스" |
| 세션 간 feedback·비자명 사실 | **memory** 파일 | Toss per-fill id 부재, codex 2채널 |
| 반복되는 프로세스 자체 | **스킬/훅** 변경 | 파이프라인 완료 게이트에 회고 |
| 없음 | — | `회고 skip(학습 없음)` 한 줄 |

**CLAUDE.md 승격 바**: CLAUDE.md는 매 세션 로드되는 가장 비싼 자리다. 아무거나 올리면 always-on 파일이 붓는다. **실제로 재발했거나(§3), 한 번이라도 고비용 실패**를 낸 학습만 규칙으로 승격한다. 대부분의 학습은 memory에 머문다.

## 5. 사람 게이트 (쓰기 무게에 비례)

- **memory 쓰기**: 자율 진행 — 메모리 시스템이 원래 그렇게 동작한다. 프리뷰 불필요.
- **CLAUDE.md·ADR·스킬/훅 쓰기**: 쓰기 *전* 프리뷰하고 승인받는다 — 규칙·결정·에이전트 동작을 바꾸고 레포에 영속되므로 무게가 다르다(§4가 "프로세스 → 스킬/훅 변경"으로 라우팅한 경우 포함).

## 동작 원칙

- **학습 없으면 없다고 한다.** 억지 통찰은 noise다. 빈 회고는 정상이고 건강하다.
- **실행 가능 > 서술.** 다음에 실행할 수 있는 한 줄이 통찰적 문단을 이긴다.
- **1개 잘 라우팅 > 10개 흐릿한 노트.**
- **self-grade 함정을 기억한다.** 내가 나를 채점하면 후하다. 흔들린 작업엔 외부 평가자를 부른다.
- **보수적으로 트리거한다.** 공격적으로 도는 회고 = 의례. description을 pushy하게 고치지 말 것 — 의도된 보수성이다.

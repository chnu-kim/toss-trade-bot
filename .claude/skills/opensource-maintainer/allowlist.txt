# opensource-maintainer 시크릿 스캔 allowlist (enforcement 설정 — CODEOWNERS 보호 대상).
#
# 형식: <레포 루트 상대 경로>|<그 줄에 포함돼야 하는 리터럴>|<사유>
#
# 매칭 규칙(의도적으로 좁다): 발견된 줄의 **경로가 정확히 일치**하고 **그 줄 내용에
# 리터럴이 포함**될 때만 그 한 줄이 제외된다. 따라서
#   - 같은 파일의 *다른* 값은 계속 검출된다(파일 전체를 여는 게 아니다).
#   - 같은 값이 *다른* 파일에 나타나면 계속 검출된다(값만으로 열리지 않는다).
#   - 디렉토리/글롭/패턴 단위 제외는 지원하지 않는다.
#
# 왜 소스 인라인 주석(`# nolint` 류)이 아니라 이 매니페스트인가:
# scan.sh는 CI에서 PR 내용을 검사하는 자동 게이트다. 인라인 마커를 전역으로 인정하면
# PR이 진짜 자격증명 옆에 마커만 달아 게이트를 통과할 수 있다(스캔 대상과 우회 수단이
# 같은 신뢰 경계 안에 있게 된다). 이 파일은 CODEOWNERS로 보호되므로 항목 추가에
# 소유자 리뷰가 필요하다 — 우회가 enforcement 설정 변경으로 승격된다.
#
# 항목을 추가하기 전에: 그 값이 정말 합성(가짜)인지 확인하라. 실제 시크릿은 절대
# allowlist 하지 말고 제거·회전한다.

# opensource-maintainer

이 레포는 **private로 유지하되 언제 public으로 전환해도 문제없는 오픈소스 품질**로 개발한다(프로젝트 표준). private라고 방심하면 시크릿·개인정보·환경 의존 내용이 커밋에 섞여, 나중에 공개가 불가능해진다. 이 스킬은 **커밋 직전에 마지막 게이트**로서 그런 내용을 걸러낸다.

## 언제 실행하나

이 프로젝트에서 커밋을 만들기 **직전**. 구체적으로:
- `git commit`을 실행하기 바로 전 (스테이징 후).
- 새 파일·문서·설정을 추가하고 올리려 할 때.

게이트이므로, 통과(또는 사용자가 항목을 확인하고 진행 결정)하기 전에는 커밋하지 않는다.

## 절차

1. **스캔 실행.** 스테이징된 변경분을 본다:
   ```bash
   bash .claude/skills/opensource-maintainer/scripts/scan.sh
   ```
   스테이징된 변경이 없으면 스크립트가 자동으로 추적 파일 전체를 본다. 전체를 강제하려면 `--all`.

2. **후보 해석.** 스크립트 출력은 *후보*다 — 기계적으로 차단하지 말고 판단한다. 아래는 정상이므로 통과시킨다:
   - **모듈 경로의 소유자명**(`github.com/<owner>/...`) — Go import 경로라 불가피하고 표준.
   - **빈 placeholder**(`configs/env.example`의 `TOSS_CLIENT_SECRET=`)나 환경변수 *이름* 참조(`os.Getenv("...")`).
   - **example류 이메일/도메인**(`user@example.com`).
   진짜 문제는 아래다:
   - 실제 값이 채워진 시크릿(키·토큰·secret), 개인키 블록, 하드코딩된 Bearer.
   - 개인 이메일(naver/gmail 등), 홈 절대경로(`/Users/...`), 비공개 지침 파일 참조(`~/.claude/...`).
   - 특정 OS/머신 가정(`macOS/arm64`, `/opt/homebrew` 등)이 추적 문서/코드에 박힌 것.
   - 실수로 추적된 `.env`.

3. **보고.** 한국어로, 카테고리별로 무엇이 왜 문제인지 간결히 보고한다. 문제가 없으면 "통과"로 명확히 알린다.

4. **수정 제안·적용.** 실제 문제가 있으면 구체적 수정안을 제시하고 적용을 제안한다(예: 개인 이메일 → GitHub noreply, 홈 경로 → 일반화/제거, 시크릿 → env 주입 + `.gitignore`, 환경 특정 문구 삭제). 적용 후 **재스캔**해 깨끗한지 확인한 뒤 커밋한다. 차단성 시크릿은 절대 그대로 커밋하지 않는다.

5. **커밋 위생 확인.** 스크립트가 `user.email`이 GitHub noreply인지, 서명이 켜져 있는지도 점검한다. noreply가 아니면 개인 이메일이 히스토리에 남으니 repo-local로 noreply 설정을 제안한다.

## 출력 형식

ALWAYS 이 구조로 보고한다:

```
## 커밋 전 점검 결과
- 차단 항목: <없음 | N건>
- 주의 항목: <없음 | N건>

### 발견 (있을 때만)
- [카테고리] 파일:줄 — 무엇/왜 + 권장 조치

### 판단
<통과: 커밋 진행 가능> 또는 <수정 필요: 아래 조치 후 재스캔>
```

문제가 없으면 발견 섹션은 생략하고 "통과"만 명확히.

## 주의

- 이 스킬은 **차단이 아니라 판단을 돕는 게이트**다. 거짓 양성(소유자명, placeholder 등)에 발목 잡혀 정상 커밋을 막지 말 것. 반대로 실제 시크릿/개인정보는 단호히 막을 것.
- 패턴을 넓히거나 좁히고 싶으면 `scripts/scan.sh`를 수정한다. 새 유형의 유출을 한 번 겪으면 패턴에 추가해 다음부터 자동으로 잡히게 한다.
